# GitHub Workflow with AI Editor

このドキュメントでは、AIエディタとGitHub CLI (`gh`) を組み合わせた効率的な開発フローを解説します。

## 1. 基本概念

### Issues (イシュー)
- **役割**: タスク管理、バグ報告、機能要望のリスト。
- **AI開発での位置づけ**:
    - 大規模な機能開発や、バグ修正の単位として作成します。
    - 個人開発の細かいタスクは、プロジェクトルートの `task.md` で管理する方が高速です。

### Pull Requests (プルリクエスト / PR)
- **役割**: コードの変更をメインブランチ（`main`）に取り込むための提案。
- **AI開発での位置づけ**:
    - **「1つのタスク = 1つのPR」** を心がけると、AIが誤ったコードを書いたときに簡単に戻せます。
    - PRを作成することで、変更内容の差分（Diff）が見やすくなり、AIによる意図しない変更に気づきやすくなります。

---

## 2. 推奨ワークフロー

AIエディタを使用する場合、以下のループを回すのが最も効率的です。

### Step 1: タスクの定義 (`task.md`)
プロジェクトルートに `task.md` を作成し、やることを箇条書きにします。
```markdown
- [ ] ログイン画面のデザイン修正
- [ ] 認証ロジックの実装
```

### Step 2: 実装とコミット
AIに指示を出し、コードを修正します。動作確認ができたらコミットします。
```bash
git add .
git commit -m "ログイン画面のデザインを修正"
```

### Step 3: プルリクエストの作成 (`gh pr create`)
ある程度まとまった機能ができたら、PRを作成します。
```bash
# ブラウザを開かずにターミナルだけで作成可能
gh pr create --title "認証機能の実装" --body "ログイン画面とロジックを追加しました。"
```
または、対話モードで作成：
```bash
gh pr create
# タイトルや本文を訊かれるので入力
```

### Step 4: マージ (`gh pr merge`)
問題なければマージして、次のタスクへ進みます。
```bash
gh pr merge --merge --delete-branch
# --merge: マージコミットを作成
# --delete-branch: 作業用ブランチを削除
```

## 3. よく使う `gh` コマンド集

| 目的 | コマンド | 説明 |
| --- | --- | --- |
| **Issue** | | |
| 一覧を見る | `gh issue list` | オープンなIssueを表示 |
| 作成する | `gh issue create` | 新しいIssueを作成 |
| 見る | `gh issue view <番号>` | Issueの詳細を表示 |
| **PR** | | |
| 一覧を見る | `gh pr list` | オープンなPRを表示 |
| 作成する | `gh pr create` | 現在のブランチからPRを作成 |
| チェックアウト | `gh pr checkout <番号>` | 他の人のPRを手元で動かす |
| マージする | `gh pr merge` | PRをマージする |
| **その他** | | |
| ブラウザで開く | `gh browse` | 現在のリポジトリをブラウザで開く |

## 4. AIへの指示のコツ
- 「`task.md` の一番上のタスクをやって」と指示すると、AIはコンテキストを理解しやすいです。
- PRを作る前に「変更内容を `git diff` で確認して」と指示すると、AI自身にセルフレビューさせることができます。
